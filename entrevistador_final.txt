LAS 12 DIMENSIONES UNIVERSALES — DETALLE COMPLETO
=======================================================================

1. ACTORES
   Define quienes interactuan con el sistema.

   Preguntas fundamentales:
   - Quien inicia acciones
   - Quien consume informacion
   - Quien supervisa
   - Quien autoriza
   - Quien automatiza

   Ejemplos:
   - usuario
   - administrador
   - sistema externo
   - sensor
   - agente IA
   - scheduler

-----------------------------------------------------------------------

2. OBJETOS
   Define que entidades existen.

   Preguntas fundamentales:
   - Que cosas persisten
   - Que datos contienen
   - Que identidad poseen
   - Que atributos son obligatorios
   - Que ciclo de vida tienen

   Ejemplos:
   - usuario
   - factura
   - orden
   - documento
   - evento
   - dispositivo

-----------------------------------------------------------------------

3. OPERACIONES
   Define acciones posibles.

   Preguntas fundamentales:
   - Que puede hacerse
   - Que transforma estados
   - Que crea objetos
   - Que destruye objetos
   - Que operaciones cruzan multiples entidades

   Tipos:
   - CRUD
   - workflow
   - batch
   - streaming
   - automatizacion
   - compensacion

-----------------------------------------------------------------------

4. ESTADOS
   Define como cambian las entidades.

   Preguntas fundamentales:
   - Estado inicial
   - Estado intermedio
   - Estado terminal
   - Estados prohibidos
   - Transiciones validas

   Reglas:
   - Todo estado requiere trigger
   - Toda transicion requiere origen y destino
   - Todo estado terminal debe explicarse

-----------------------------------------------------------------------

5. REGLAS
   Define restricciones universales.

   Preguntas fundamentales:
   - Que nunca debe ocurrir
   - Que validaciones existen
   - Que restricciones legales existen
   - Que politicas existen
   - Que condiciones invalidan operaciones

   Tipos:
   - negocio
   - seguridad
   - consistencia
   - cumplimiento
   - integridad

-----------------------------------------------------------------------

6. EVENTOS
   Define que sucede externamente.

   Preguntas fundamentales:
   - Que dispara acciones
   - Que sucede fuera del sistema
   - Que eventos temporales existen
   - Que eventos fallidos existen
   - Que eventos requieren reaccion inmediata

   Tipos:
   - usuario
   - tiempo
   - sistema externo
   - sensor
   - webhook
   - error

-----------------------------------------------------------------------

7. TIEMPO
   Define comportamiento temporal.

   Preguntas fundamentales:
   - Cuando ocurre algo
   - Cuanto tarda
   - Que expira
   - Que depende del orden temporal
   - Que ventanas existen

   Tipos:
   - timeout
   - SLA
   - expiracion
   - scheduler
   - ventana de mantenimiento

-----------------------------------------------------------------------

8. RECURSOS
   Define limites y consumo.

   Preguntas fundamentales:
   - Que consume el sistema
   - Que limites existen
   - Que costos existen
   - Que se agota
   - Que escalabilidad requiere

   Tipos:
   - CPU
   - memoria
   - almacenamiento
   - ancho de banda
   - APIs externas
   - operadores humanos

-----------------------------------------------------------------------

9. RELACIONES  [DIMENSION ADITIVA V6]
   Define como se conectan entidades y actores.

   Preguntas fundamentales:
   - Quien pertenece a quien
   - Que depende de que
   - Que cardinalidad existe
   - Que herencia existe
   - Que composicion existe
   - Que propagacion existe

   Tipos:
   - 1:1
   - 1:N
   - N:N
   - dependencia
   - ownership
   - composicion

-----------------------------------------------------------------------

10. ESPACIO  [DIMENSION ADITIVA V6]
   Define localizacion y distribucion.

   Preguntas fundamentales:
   - Donde ocurre algo
   - Que esta distribuido
   - Que regiones existen
   - Que componentes estan cerca o lejos
   - Que latencias geograficas existen

   Tipos:
   - local
   - edge
   - cloud
   - multi-region
   - on-premise
   - hibrido

-----------------------------------------------------------------------

11. COMUNICACION  [DIMENSION ADITIVA V6]

   Preguntas fundamentales:
   - Como se comunican componentes
   - Que protocolos usan

   - Que sincronizacion existe

   Tipos:
   - sync
   - async
   - pub/sub
   - queue
   - websocket
   - RPC
   - REST
   - GraphQL

-----------------------------------------------------------------------

12. OBJETIVOS  [DIMENSION ADITIVA V6]

   - Que busca optimizar
   - Que problema resuelve
   - Como se mide el exito

   - Que tradeoffs acepta

   Tipos:
   - reducir tiempo
   - reducir costo
   - aumentar throughput
